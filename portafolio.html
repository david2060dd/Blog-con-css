Proyecto 1


1.-Creacion de una aplicación de Tienda en el Framework QT (C++)

Descripción:

Con todo lo aprendido en QT(Herramientas de interfaz gráfica).Construir una aplicación Que sirva para gestionar los productos de una tienda y que a su vez sirva como programa de facturación


Detalles
Ventana
  1. Espacios para agregar el nombre, cedula, teléfono, dirección y el e-mail
  2. El dueño pueda seleccionar un producto y automáticamente salga el costo del mismo
  3. Espacio para colocar la cantidad del producto
  4. Visualizar una tabla con: código del producto, nombre del producto, cantidad del producto

producto
  1. Espacio para visualizar el subtotal, IVA y total
  2. Botón para finalizar la compra
  3. Validar cedula
  4. Validar todos los datos al momento de pulsar el botón de finalizar
  5. Soltar una “factura” al momento de tocar el botón de finalizar
  6. Al pulsar en el botón de aceptar la ventana de la factura de debe cerrar y limpiar lodos los datos ingresados del programa
  7. Tener opción para agregar más productos

Proyecto 2


2.-Aplicación de dibujo

Descripción:

Crear un utilitario que permita dibujar figuras geométricas básicas (circunferencias, rectas y rectángulos). El nombre para este utilitario es Paint++. Y difiere de otros utilitarios en el hecho de que las figuras dibujadas pueden almacenarse dentro de una base de datos, para recuperarse posteriormente y volverse a dibujar


Detalles

  1. Nuevo: Borra el área de dibujo y todos los datos del programa (no de la base de datos).
  2. Abrir: Carga información desde una base de datos y dibuja las figuras en el área de dibujo.
  3. Guardar: Guarda las figuras actuales en la base de datos.
  4. Línea: Permite dibujar una línea sobre el área de dibujo.
  5. Rectángulo: Permite dibujar un rectángulo sobre el área de dibujo.
  6. Circulo: Permite dibujar un círculo sobre el área de dibujo.
  7. Color: Permite seleccionar el color con el que se desea dibujar (JColor Choicer).
  8. Barra de Menús: Contiene todas las acciones de la barra de herramientas.
  9. Área de dibujo: Permite dibujar las figuras geométricas utilizando el ratón
  10. Simulación: Permite recrear todos los elementos generados de forma secuencial

Proyecto 3


3.-Aplicación para Cine

Descripción:

Desarrollar una aplicación tipo “Wizard” (Asistente) done tengamos la opción de poder escoger entre dos salas, salas de cine donde se presentan dos tipos de eventos: películas y conferencias. También se requiere incorporar un menú para que los clientes puedan realizar las compras de boletos sin necesidad de pasar por una boletería. Para ello requieren de una aplicación de escritorio con interfaz gráfica, minimalista e intuitiva que permita a sus clientes manejarse sin dificultad ni asistencia de ningún empleado.


Detalles

Selección de evento:

En la primera pantalla aparecerá una cartelera de los Eventos que se están presentando en las salas de cine. Para ello el usuario deberá seleccionar entre Conferencia o Película. Todos los eventos comparten las siguientes características: Póster (Imagen del evento), Título y Duración (tiempo). Las películas tienen los siguientes atributos: Género, Censura y Sinápsis. Las conferencias tienen los siguientes atributos: Presentador y Temática. Cada evento dispone de varios Horarios donde se establece la hora (HH:MM) que se presenta el evento. Los eventos se desarrollan en diferentes Salas de cine del local. Cada sala dispone de un Número y una propiedad que le permite diferenciar salas de cine regulares de salas de cine IMAX.

El usuario debe seleccionar para que tipo de evento quiere comprar los boletos (película o conferencia). Una vez que realizada esta acción se cargará una lista desplegable con los títulos del tipo de evento seleccionado. Cada vez que el usuario cambie el título del evento, se cargará sobre la misma pantalla el póster (imagen) del evento, el número de la sala donde se proyecta, si la sala es IMAX o no y una lista de los horarios disponibles. Adicional, existirá un botón denominado “Detalles” que mostrará un cuadro de diálogo con los detalles del evento (detalles propios de una película o de una conferencia). Una vez que el usuario haya elegido el evento que le interesa, deberá seleccionar un horario de la lista. Solo cuando esto haya sucedido, se activará un botón denominado “Siguiente” para continuar a la siguiente etapa. Si el usuario, selecciona otro título, o cambia el tipo de evento los controles deben volver a un estado de inicio (vacíos y desactivados).


Compra de boletos

La segunda pantalla se ingresa los datos del Cliente (Cédula, Nombre, Teléfono, Dirección) y los datos del Boleto (Número de adultos y Número de niños) para el evento, así como también el detalle de los valores a cancelar (SubTotal, IVA y Total) La pantalla mostrará el tipo de evento seleccionado en la pantalla anterior y su título. Adicional, los campos de texto para ingresar el número de adultos y niños que desean ingresar al evento. Los valores a cancelar se calcularán considerando las siguientes restricciones:


  1. Precio boleto de película = $ 5.60
  2. Precio boleto de conferencia = $ 9.30
  3. Los niños pagan el 50% del precio del boleto
  4. Si la película se proyecta en una sala IMAX, el precio por boleto se incrementa un 20%

Si la película tiene censura para menores de 12 años, el campo de texto para ingresar el número de boletos para niños se deshabilitará. Solamente cuando se haya ingresado al menos un número de boletos, se activará un botón denominado “Siguiente” para continuar a la siguiente etapa. Se dispondrá de un botón denominado “Cancelar” que cancelará toda la operación y mostrará la pantalla inicial en su estado de inicio.


Confirmación

Dentro de esta pantalla el usuario mirá un resumen de la compra que está realizando. Se tendrá el título del evento, la sala, el número de boletos (adultos y niños), los asientos y los detalles de los valores a cancelar. Dentro de la pantalla existirá un botón denominado “Finalizar”, el cual mostrará un mensaje de diálogo agradeciendo al usuario (mostrar el nombre) por la compra realizada. Cuando el cliente cierre la ventana, la aplicación regresará a la pantalla inicial en su estado de inicio. Aquí también estará presente el botón “Cancelar” de la pantalla anterior


Restricciones
  1. La aplicación se desplegará en monitores “touch” con resolución 1024 x 768 píxeles
  2. Se debe utilizar clases y métodos genéricos
  3. Se debe implementar variables de tipo args y autoboxing
  4. Deben considerarse todos los posibles errores que un cliente pueda cometer (Utilizar ER)
  5. Los datos de los eventos estarán quemados dentro del código de la aplicación utilizar una clase de tipo ENUM
  6. Los asientos de cada sala deben estar predeterminados, y se debe ir consumiendo según la compra de cada cliente
  7. Debe existir un punto donde al momento de comprar se diga que los boletos están agotados
  8. Debe tener una arquitectura MVC
  9. Fecha de entrega: 18/12/2020 - Hora de entrega: 07:00 – 10:00
